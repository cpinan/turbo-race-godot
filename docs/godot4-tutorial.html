Chapter 01

GDScript — Static Typing

GDScript is Python-like but compiled. Use static types everywhere — catches bugs at editor-time, generates faster bytecode, and matches the safety of C++.

# BAD — dynamic, no type safety
var speed = 11.0
func move(delta):
    position.x += speed * delta
# GOOD — static typing throughout
var speed: float = 11.0

func move(delta: float) -> void:
    position.x += speed * delta

Key GDScript types:

TypeC++ equivalent
floatfloat
intint
boolbool
Stringstd::string
Vector2Vec2
Rect2bounding-box struct
Array[T]std::vector<T>
Dictionarystd::map
Rule: Every var, every function parameter, every return type gets a type annotation. No exceptions.

Chapter 02

Scene Tree & Nodes

Everything in Godot is a Node in a tree. Scenes are reusable subtrees saved as .tscn files. Scripts attach to nodes with extends.

Main (Node) ├── GameScene (Node2D) │ ├── Player (CharacterBody2D) │ ├── Parallax (Node2D) │ └── ObstacleRoot (Node2D) ├── HUD (CanvasLayer) │ ├── ScoreLabel (Label) │ └── JoystickBG (TextureRect) ├── PauseScreen (CanvasLayer) └── GameOverScreen (CanvasLayer) 
# Script attaches to a node
extends CharacterBody2D
class_name BaseVehicle

func _ready() -> void:
    # called when node enters the scene tree
    pass

Node lifecycle:

  1. _ready() — node entered tree, all children ready
  2. _process(delta) — every frame (rendering)
  3. _physics_process(delta) — fixed rate (physics)
  4. _input(event) — raw input events
  5. _unhandled_input(event) — input not consumed

Accessing child nodes:

# By path ($shortcut = get_node)
@onready var _label: Label = $ScoreLabel
@onready var _btn: TextureButton = $BtnPause

# @onready defers until _ready() fires
# — safe to use in _ready() and beyond

Chapter 03

Coordinate System — The Y-Axis Problem

Cocos2d-x: Y-up (origin bottom-left)
Godot 2D: Y-down (origin top-left)

Cocos2d-x Godot 2D (0,768) ●─────── ●───────── (1024,0) │ Y↑ Y↓ │ │ │ (0,0) ●──────→ X X ←────● (0,768)
If you port positions directly without compensating, every sprite appears upside-down and Y coordinates are mirrored.

Solution: flip the root node

# GameScene root node (in .tscn or _ready)
# Flips the entire coordinate space to Y-up
scale    = Vector2(1, -1)
position = Vector2(0, 768)  # WIN_H
# Each Sprite2D inside needs a counter-flip
# (otherwise texture appears upside-down)
$PlayerSprite.scale = Vector2(1, -1)
CanvasLayer is exempt. It bypasses all parent transforms — always use Y-down coordinates inside CanvasLayer nodes (UI, HUD).
With this flip, every C++ position value works unchanged in Godot. position.y = 300 still means 300 units from the bottom.

Chapter 04

CanvasLayer — UI That Ignores World Transforms

A CanvasLayer node renders its children in a separate 2D canvas, ignoring any transforms from parent nodes — including the Y-flip on GameScene.

# HUD stays in screen space regardless of
# what the game world does
class_name HUD
extends CanvasLayer

@onready var _score: Label = $ScoreLabel

func show_hud() -> void:
    visible = true

func hide_hud() -> void:
    visible = false

The layer property controls draw order (higher = in front of lower layers).

When to use CanvasLayer:

  • All HUD elements (score, pause button)
  • Pause screen, game-over screen, home screen
  • Virtual joystick overlay
  • Any UI that must stay fixed to the screen

When NOT to use CanvasLayer:

  • Game world objects (player, obstacles, parallax)
  • Anything that should move with the game camera
Think of CanvasLayer like Cocos2d-x ui::Layer — it's always in "screen space", not "world space".

Chapter 05

_process vs _physics_process

# _process: runs every rendered frame
# delta = time since last frame (variable)
func _process(delta: float) -> void:
    # Good for: animations, UI, camera
    _update_score_label()
# _physics_process: fixed timestep
# Default: 60 Hz (ProjectSettings)
func _physics_process(delta: float) -> void:
    # Good for: movement, collision, input
    _move_player(delta)
    _scroll_world(delta)
    _check_collisions()
Physics and movement go in _physics_process. Mixing them in _process causes frame-rate-dependent speed.
_process_physics_process
RateVariable (fps)Fixed (60 Hz)
deltaVariesConstant (~0.0167)
Use forVisuals, UIPhysics, input
Cocos2d-x equivupdate()physics step

Cocos2d-x update(dt) maps to _physics_process in Godot — not _process. The original game ran collision checks and movement inside a fixed-tick scheduler.

# Godot constant for physics FPS
const PHYSICS_FPS: float = 60.0
# Use when converting speed from px/tick to px/s:
# speed_px_per_s = speed_px_per_tick * PHYSICS_FPS

Chapter 06

Signals — Godot's Event System

Signals replace Cocos2d-x notification strings (CC_BROADCAST_MESSAGE_TYPE_GAME_OVER) with type-safe, decoupled events.

# Define a signal
class_name GameManager
extends Node

signal game_over
signal score_changed(total: int)

# Emit
func trigger_game_over() -> void:
    emit_signal("game_over")

# Emit with data
func add_score(n: int) -> void:
    _total += n
    emit_signal("score_changed", _total)
# Connect in another node
func _ready() -> void:
    GameManager.game_over.connect(_on_game_over)
    GameManager.score_changed.connect(_on_score)

func _on_game_over() -> void:
    show_game_over_screen()

func _on_score(total: int) -> void:
    _label.text = str(total)
Prefer signals over polling. Don't check a global "is_game_over" flag every frame — emit a signal once when it happens.

Lambdas also work for short handlers:

_btn.pressed.connect(
    func(): emit_signal("pause_pressed")
)

Chapter 07

Autoloads — Global Singletons

Autoloads are scripts that Godot instantiates at startup and keeps alive for the entire session. They're accessible from any script by name — like C++ global managers.

Registered in Project → Project Settings → Autoload (or in project.godot):

[autoload]
SaveManager    = "*res://autoload/save_manager.gd"
AudioManager   = "*res://autoload/audio_manager.gd"
GameManager    = "*res://autoload/game_manager.gd"
ScoreModel     = "*res://autoload/score_model.gd"
LevelLoader    = "*res://autoload/level_loader.gd"
LeaderboardService = "*res://autoload/leaderboard_service.gd"
# Use from anywhere — no import needed
AudioManager.play_music()
ScoreModel.add_obstacle_avoided()
if SaveManager.is_muted():
    AudioManager.mute()
AutoloadResponsibility
GameManagerState machine, game_over signal
ScoreModelObstacle count × 100, score_changed signal
AudioManagerMusic rotation, SFX, mute
SaveManageruser://save.json, best scores, settings
LevelLoaderJSON → LevelData cache
LeaderboardServiceGoogle Play Games, no-op on non-Android
Don't put game logic in autoloads. They hold state and emit signals — not update loops.

Chapter 08

Pure Physics Functions — Testable by Design

The most important architectural rule: all physics and collision logic must be pure static functions with no Node or scene-tree dependency.

# scripts/physics/vehicle_physics.gd
class_name VehiclePhysics
# No "extends" — not a Node at all!

const MAX_PLAYER_JUMP: float = 140.0
const JUMP_DURATION:   float = 0.6
const DEFAULT_SPEED:   float = 11.0

static func jump_arc_offset(t: float) -> float:
    return MAX_PLAYER_JUMP * sin(t * PI)

static func can_jump(
        airborne_h: float,
        is_jumping: bool) -> bool:
    return airborne_h <= 1.0 and not is_jumping

Why pure functions?

  • Unit-testable without a running Godot editor
  • No mock nodes, no scene setup in tests
  • Logic is isolated from rendering/timing
  • Easy to port: same inputs → same outputs as C++

The node calls the pure function:

# In base_vehicle.gd (the Node)
func do_jump() -> void:
    if not VehiclePhysics.can_jump(
            _airborne_height, _is_jumping):
        return
    _is_jumping = true
    _jump_t     = 0.0
The node handles timing and state; the pure function handles math. Never the other way around.

Chapter 09

Jump Arc — Sine, Not Parabola

Cocos2d-x JumpBy::create(0.6f, Vec2::ZERO, 140.0f, 1) uses:

y = height × |sin(π × t)| 
# CORRECT — matches Cocos2d-x JumpBy
static func jump_arc_offset(t: float) -> float:
    return MAX_PLAYER_JUMP * sin(t * PI)

# WRONG — parabola, do not use
static func jump_arc_offset(t: float) -> float:
    return MAX_PLAYER_JUMP * 4.0 * t * (1.0 - t)
Both curves peak at the same height. The difference is when they hit the threshold for air-obstacle collision (~63 units). Sine: t≈0.149. Parabola: t≈0.129. Wrong formula = wrong difficulty.

Driving the arc in _physics_process:

var _jump_t:     float = 0.0
var _is_jumping: bool  = false
var _player_y:   float = 0.0  # ground level

func _physics_process(delta: float) -> void:
    if _is_jumping:
        _jump_t += delta / VehiclePhysics.JUMP_DURATION
        if _jump_t >= 1.0:
            _jump_t     = 0.0
            _is_jumping = false
            position.y  = _player_y
        else:
            position.y = _player_y + \
                VehiclePhysics.jump_arc_offset(_jump_t)

Constants from C++ BaseVehicle.hpp:

C++GDScript
MAX_PLAYER_JUMP = 140.0fMAX_PLAYER_JUMP: float = 140.0
JUMP_DURATION = 0.6fJUMP_DURATION: float = 0.6
jumps = 1single arc, no bounce

Chapter 10

Collision Rectangles

Godot has built-in physics bodies, but for this game type manual rect intersection is more precise and easier to port from C++.

All C++ collision rects are expressed as fractions of contentSize. Port them exactly — never guess.

# From BaseVehicle::getGroundCollision()
static func ground_collision_rect(
        pos_x: float, player_y: float,
        w: float, h: float) -> Rect2:
    return Rect2(
        pos_x - w*0.5 + w*0.3,  # x: 30% inset
        player_y,                   # y: ground level
        w * 0.55,                  # width:  55% of sprite
        h * 0.3                    # height: 30% of sprite
    )
# From BaseVehicle::getAirCollision()
# NOTE: height dimension uses WIDTH, not height
# This is intentional in the C++ source!
static func air_collision_rect(
        pos_x: float, pos_y: float,
        w: float, h: float) -> Rect2:
    return Rect2(
        pos_x - w*0.5 + w*0.3,
        pos_y - h*0.5 + h*0.16,
        w * 0.55,
        w * 0.2   # uses w intentionally!
    )

Intersection check:

if vehicle_rect.intersects(obstacle_rect):
    vehicle.die()
Read every collision rect from the original C++ source. Do not approximate or "eyeball" them — tiny differences change perceived fairness of the game.

Chapter 11

Obstacle Collision Guards

Each obstacle type has a guard check that runs before the rect intersection. If the guard returns false, the collision is skipped — even if rects overlap.

SingleObstacle
Lane-band wall

Player Y must be within the lane's vertical band.

# Guard
var y_eff = player_y + h * 0.15
if y_eff < top or y_eff > top + h*0.37:
    return false  # wrong lane
# then check rect

DoubleObstacle
Ground jump

No guard — always collidable. Uses two collision rects (one per pillar).

# No guard, just rect check
var r1 = rect_left(obs)
var r2 = rect_right(obs)
return veh_rect.intersects(r1) \
    or veh_rect.intersects(r2)

AirDoubleObstacle
Air obstacle

Player must be jumping AND above the height threshold.

# Guard 1: must be jumping
if not is_jumping: return false
# Guard 2: high enough
if airborne_h < 63.0: return false
# 63.0 = MAX_JUMP * 0.45
The 63-unit threshold (MAX_PLAYER_JUMP × 0.45) comes from AirDoubleObstacle.cpp. It means the player is only at risk during the top 55% of the jump arc — punishing reckless high jumps, not accidental ones.

Chapter 12

Z-Depth Ordering

Cocos2d-x uses setLocalZOrder(z). Godot uses z_index on Node2D — higher value renders in front.

The C++ GameDeep enum defines z values per layer. Convert using:

z_index = int((WIN_H − z_param) / 10)

where WIN_H = 768 and z_param is the Cocos z value. Higher z_param → lower z_index → behind.

LayerC++ z_paramGodot z_index
Air obstacle (front)076
Player (dynamic)lane_y ≈ 380~46
Single obstaclelane_y39–46
Ground obstacle (back)WIN_H × 0.5 = 38438

Player z_index must update every frame:

func _physics_process(_delta: float) -> void:
    var z_param := _player.player_y \
        + _player.content_size.y * 0.75
    _player.z_index = int(
        (WIN_H - z_param) / 10.0)
If you add obstacles as children AFTER the player in the scene tree and don't set z_index, all obstacles render in front of the player. Always set z_index explicitly at spawn time.

Chapter 13

Touch & Mouse Input

Godot delivers input as events. Handle them in _unhandled_input to avoid consuming events that UI nodes should receive first.

func _unhandled_input(event: InputEvent) -> void:
    # Mobile touch
    if event is InputEventScreenTouch:
        _handle_touch(event)

    # Mobile drag (while touching)
    elif event is InputEventScreenDrag:
        _handle_drag(event)

    # PC left mouse button
    elif event is InputEventMouseButton \
         and event.button_index == MOUSE_BUTTON_LEFT:
        _handle_mouse_btn(event)

    # PC mouse move (while pressed)
    elif event is InputEventMouseMotion:
        _handle_mouse_move(event)

Screen split: left = joystick, right = jump

const WIN_W: float = 1024.0

func _handle_touch(e: InputEventScreenTouch) -> void:
    var left := e.position.x < WIN_W * 0.5
    if left and e.pressed:
        _start_joystick(e.position, e.index)
    elif e.index == _joy_index and not e.pressed:
        _stop_joystick()
    elif not left and e.pressed:
        _player.do_jump()
Use event.index (finger ID) to distinguish simultaneous touches. Track which finger owns the joystick — ignore drags from other fingers.

Mouse mode: store _joy_index = -1 to signal "mouse, not touch". Check _joy_index == -1 in MouseMotion handler to avoid acting on stale state.

Chapter 14

Virtual Joystick Pattern

The virtual joystick is split across two nodes: game_scene.gd handles physics input (movement), hud.gd handles the thumb visual independently.

# game_scene.gd — physics logic
const JOY_MAX_DIST:  float = 80.0
const JOY_DEAD_ZONE: float = 20.0

var _joy_active:    bool  = false
var _joy_index:     int   = -1  # -1 = mouse
var _joy_anchor_x:  float = 0.0
var _joy_anchor_y:  float = 0.0
var _joy_norm_x:    float = 0.0  # -1..1
var _joy_norm_y:    float = 0.0  # -1..1

func _update_joy(pos: Vector2) -> void:
    var dx := pos.x - _joy_anchor_x
    var dy := pos.y - _joy_anchor_y
    _joy_norm_x = clampf(dx / JOY_MAX_DIST, -1, 1)
    _joy_norm_y = clampf(-dy / JOY_MAX_DIST, -1, 1)
    if absf(dx) < JOY_DEAD_ZONE: _joy_norm_x = 0.0
    if absf(dy) < JOY_DEAD_ZONE: _joy_norm_y = 0.0
# Apply in _physics_process
if _joy_active:
    var spd := VehiclePhysics.DEFAULT_SPEED \
               * delta * PHYSICS_FPS
    var vel := Vector2(
        _joy_norm_x * spd,
        _joy_norm_y * spd)
    _player.do_move(vel, WIN_W)
Pass both X AND Y to do_move(). The original C++ HudLayer::updateControl calls doMove(Vec2) with a full 2D vector. Passing only Y means the player can't move left/right. 
# hud.gd — thumb visual (independent)
const THUMB_REST: Vector2 = Vector2(45, 45)
const JOY_RADIUS: float   = 45.0

func _move_thumb(dx: float, dy: float) -> void:
    var nx := clampf(dx / JOY_MAX_DIST, -1, 1)
    var ny := clampf(dy / JOY_MAX_DIST, -1, 1)
    _joy_thumb.position = THUMB_REST + \
        Vector2(nx * JOY_RADIUS, ny * JOY_RADIUS)

Chapter 15

Parallax Scrolling

Cocos2d-x scrolls multiple background layers at different speeds (MAX_PARALLAX = 5). Port this as an array of Sprite2D nodes scrolled manually each frame.

# 5 layers, slowest (sky) to fastest (track)
const PARALLAX_SPEEDS: Array[float] = \
    [0.1, 0.2, 0.4, 0.6, 1.0]

@onready var _layers: Array[Sprite2D] = [
    $BG0, $BG1, $BG2, $BG3, $BG4
]

func _scroll_parallax(
        world_speed: float,
        delta: float) -> void:
    for i in range(_layers.size()):
        var layer := _layers[i]
        layer.position.x -= world_speed \
            * PARALLAX_SPEEDS[i] * delta
        # Seamless wrap
        if layer.position.x < -WIN_W:
            layer.position.x += WIN_W * 2.0

Why each layer needs two copies:

BG0 [──── copy A ────][──── copy B ────] ↑ viewport ↑ When A scrolls off left edge: A.position.x += WIN_W * 2 ← wraps to right

Godot has a built-in ParallaxBackground node, but manual scrolling is simpler to control and easier to port 1:1 from C++.

Godot alternative (built-in):

# ParallaxBackground → ParallaxLayer
# motion_scale controls speed ratio
$ParallaxLayer.motion_scale = Vector2(0.1, 0)
# Simpler but less direct mapping from C++

Chapter 16

Object Pooling

Instantiating nodes mid-game causes hitches. Pool and recycle — same pattern as Cocos2d-x ObstaclePool<T>.

# scripts/data/obstacle_pool.gd
class_name ObstaclePool

var _scene:  PackedScene
var _parent: Node
var _pool:   Array[BaseObstacle] = []

func init(scene: PackedScene,
           parent: Node) -> void:
    _scene  = scene
    _parent = parent

func acquire() -> BaseObstacle:
    for obs in _pool:
        if not obs.visible:
            obs.visible = true
            return obs
    # Pool exhausted — grow it
    var obs: BaseObstacle = _scene.instantiate()
    _parent.add_child(obs)
    _pool.append(obs)
    return obs
func recycle(obs: BaseObstacle) -> void:
    obs.visible = false

func recycle_all() -> void:
    for obs in _pool:
        obs.visible = false

Three separate pools — one per type:

# In game_scene.gd
var _single_pool: ObstaclePool
var _double_pool: ObstaclePool
var _air_pool:    ObstaclePool

func _ready() -> void:
    _single_pool = ObstaclePool.new()
    _single_pool.init(
        preload("res://scenes/obstacles/single_obstacle.tscn"),
        $ObstacleRoot)
Recycle means visible = false, not queue_free(). The node stays in the scene tree and gets reused next spawn.

Chapter 17

Scoring & World Speed

Scoring (from ScoreModel.hpp):

const K_SCORE_FACTOR: float = 100.0

var _obstacles_avoided: int = 0

func add_obstacle_avoided() -> void:
    _obstacles_avoided += 1
    emit_signal("score_changed",
        get_total_score())

func get_total_score() -> int:
    return int(_obstacles_avoided * K_SCORE_FACTOR)
Track count of obstacles avoided, not distance or time. Multiply by 100 only for display. This matches C++ exactly.

World speed (from GameLayer.cpp):

# From Constants.h
const START_WORLD_SPEED:  float = 512.0
# designWidth * 0.5  (1024 * 0.5)
const MIN_DISTANCE:       float = 568.89
# designWidth / 1.8
const START_X_OBSTACLES:  float = 1945.6
# designWidth * 1.9

# Advance each frame
func _advance_world(delta: float) -> void:
    _world_speed += \
        _level.speed_increase * delta
    _world_speed = minf(
        _world_speed, _level.max_speed)
All speed constants derive from designWidth = 1024. Never hardcode the arithmetic result without a comment showing the formula.

Chapter 18

Testing with GUT

GUT (Godot Unit Test) is a test framework vendored in addons/gut/. Tests are GDScript files with names starting test_.

# tests/unit/test_vehicle_physics.gd
extends "res://addons/gut/test.gd"

func test_jump_arc_at_midpoint() -> void:
    var y := VehiclePhysics.jump_arc_offset(0.5)
    # sin(π/2) = 1.0 → peak = MAX_JUMP
    assert_almost_eq(y, 140.0, 0.001)

func test_cannot_jump_when_airborne() -> void:
    assert_false(VehiclePhysics.can_jump(10.0, false))

func test_can_jump_when_grounded() -> void:
    assert_true(VehiclePhysics.can_jump(0.5, false))

func test_air_guard_below_threshold() -> void:
    # 50 < 63 → should NOT collide
    assert_false(ObstaclePhysics.check_air_guard(
        50.0, true))

Run headless (CI / terminal):

godot --headless --path . \
  -s addons/gut/gut_cmdln.gd \
  -gdir=res://tests \
  -gprefix=test_ \
  -ginclude_subdirs \
  -gexit

Run a single file:

godot --headless --path . \
  -s addons/gut/gut_cmdln.gd \
  -gtest=res://tests/unit/test_vehicle_physics.gd \
  -gexit

GUT assert methods:

Chapter 19

Android Export & Leaderboards

Google Play Games Services runs only on Android. The Godot plugin is loaded at runtime — always check availability before calling any method.

# autoload/leaderboard_service.gd
var _available: bool = false
var _signed_in: bool = false

func _ready() -> void:
    _available = Engine.has_singleton(
        "GodotPlayGameServices")
    if _available:
        _try_sign_in()

func submit_score(
        leaderboard_id: String,
        score: int) -> void:
    if not _available or not _signed_in:
        return  # fail silently — never block gameplay
    var plugin = Engine.get_singleton(
        "GodotPlayGameServices")
    plugin.submitScore(leaderboard_id, score)

Key principles:

  • Leaderboard / achievement IDs are public identifiers — safe to commit, same as in C++ Constants.h
  • A sign-in failure must never block gameplay — always fail silently
  • Use Engine.has_singleton() to detect plugin availability — works on any platform
  • iOS / Game Center: keep the interface platform-agnostic; don't build an iOS backend now

Level → leaderboard mapping:

func submit_score_for_level(
        level: String, score: int) -> void:
    match level:
        "easy":   submit_score(LEAD_EASY_MODE,   score)
        "normal": submit_score(LEAD_NORMAL_MODE, score)
        "hard":   submit_score(LEAD_HARD_MODE,   score)

Chapter 20

Parity Checklist

Before calling any ported system complete, verify all of these:

Physics

  • All constants read from C++ source — not guessed
  • Jump arc uses sin(πt), not 4t(1-t)
  • Air obstacle guard threshold = MAX_JUMP × 0.45 = 63.0
  • Collision rect fractions match C++ contentSize fractions
  • SingleObstacle lane-band guard applied before rect check

Rendering

  • Root node has scale=(1,-1), position=(0,WIN_H)
  • All Sprite2D children counter-flipped
  • Z-index formula applied at spawn: int((WIN_H-z)/10)
  • Player z_index updated every frame from current Y
  • UI on CanvasLayer — not inside Y-flipped game world

Input & Gameplay

  • Joystick passes both X and Y to do_move(Vector2)
  • Touch and mouse both handled in _unhandled_input
  • Joystick thumb visual moves in HUD independently
  • Scoring = obstacles_avoided × 100, not distance
  • World speed starts at 512.0, increases per level JSON

Tests

  • Pure physics functions have unit tests
  • Golden-run regression tests cover all 3 obstacle types
  • All tests pass headless: godot --headless … -gexit
  • CI runs tests on every push
  • No test mocks game world nodes — pure function tests only
